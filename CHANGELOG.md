# Changelog



All notable changes to this project will be documented in this file.



This project follows a simple versioning model:



| Version | Meaning |

|---|---|

| `0.x` | Experimental drafts |

| `1.0` | First stable public version |

| `1.x` | Backward-compatible improvements |

| `2.0` | Major structural revision |



The format is inspired by Keep a Changelog, adapted to the needs of an educational documentation standard.

## [1.0.0] - 2026-XX-XX



### Added



- Initial public specification of **PEDAGOGY.md**.

- Definition of **PEDAGOGY.md** as an open documentation standard for pedagogical design and AI-supported education.

- Advanced 18-section structure for course-level pedagogical specification.

- Distinction between:

  - `PEDAGOGY.md` as the standard specification;

  - `PEDAGOGY.template.md` as the editable teacher template;

  - `PEDAGOGY.example.md` as a completed example.

- Initial `README.md` for the GitHub repository.

- Initial `PEDAGOGY.md` specification file.

- Initial `PEDAGOGY.template.md` file.

- Initial `PEDAGOGY.example.md` file.

- Initial `docs/getting-started.md` guide.

- Initial `docs/ai-usage.md` guide.

- Initial `docs/file-ecosystem.md` guide.

- Initial `CONTRIBUTING.md`.

- Initial `ROADMAP.md`.

- Initial `CHANGELOG.md`.



### Added — Core structure



The v1.0 structure includes the following sections:



1. Course Identity

2. Educational Context

3. Learner Profile

4. Learning Outcomes and Competences

5. Pedagogical Principles

6. Methodological Approach

7. Content Organization

8. Learning Activities

9. Assessment Design

10. Feedback Model

11. Quality Criteria for Student Work

12. Roles and Responsibilities

13. AI Use Policy

14. AI Agent Behaviour

15. Inclusion and Accessibility

16. Learning Evidence and Analytics

17. Related Files and Modular Extensions

18. Versioning and Maintenance



### Added — Documentation



- Quick-start guidance for teachers and teaching teams.

- Guidance on how AI systems should use **PEDAGOGY.md**.

- Explanation of the modular file ecosystem.

- Contribution principles and guidelines.

- Roadmap for future versions.



### Added — AI guidance



- Defined **PEDAGOGY.md** as the primary pedagogical context for AI systems.

- Added guidance for AI-generated:

  - activities;

  - rubrics;

  - explanations;

  - formative feedback;

  - student support;

  - teacher support;

  - AI agent configuration;

  - learning evidence interpretation.

- Added refusal and redirection rules for:

  - academic integrity;

  - assessment authority;

  - sensitive data;

  - contradictions with course AI policy;

  - missing pedagogical context.

- Added base system instructions for AI assistants using **PEDAGOGY.md**.



### Added — Modular ecosystem



Defined the relationship between **PEDAGOGY.md** and complementary files:



- `assessment.md`;

- `activities.md`;

- `content.md`;

- `ai-policy.md`;

- `feedback.md`;

- `cases.md`;

- `analytics.md`;

- `ethics.md`;

- `rubrics/`;

- `resources/`;

- `prompts/`;

- `agents/`;

- `skills/`.



### Added — Contribution model



- Added contribution principles:

  - pedagogical clarity;

  - teacher usability;

  - instructional alignment;

  - modularity;

  - AI-readability;

  - ethical responsibility;

  - institutional adaptability.

- Added guidance for proposing:

  - new fields;

  - new sections;

  - complementary files;

  - examples;

  - documentation improvements.



### Known limitations



- No formal JSON schema is included in v1.0.

- No automated validation tool is included.

- No complete multilingual documentation is included.

- No LMS integration is included.

- No full research background document is included in the v1.0 release.

- No official issue templates are included in v1.0.

- No official pull request template is included in v1.0.

- No advanced agent library is included in v1.0.

- No complete skill system is included in v1.0.
